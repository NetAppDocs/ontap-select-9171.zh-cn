= 了解ONTAP Select数据保护和效率
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
ONTAP Select基于ONTAP存储软件构建，通过高可用性、无共享的横向扩展架构高效地提供企业存储服务。您可以部署包含一个、两个、四个、六个或八个节点的解决方案，每个节点的 NFS、SMB/CIFS 和 iSCSI 连接存储的原始容量高达 400 TB。您可以利用原生重复数据删除和压缩功能来增加有效容量，从而降低存储成本。横向扩展架构支持高可用性和无中断数据移动，以实现负载平衡或维护硬件。



== 快照数据保护

ONTAP Select包含数据保护功能，包括快照和SnapMirror软件。您可以快速将数据复制到其他ONTAP存储，无论是在本地、远程站点还是在云端。如果您必须快速恢复数据， SnapRestore软件可以使用本地快照在几秒钟内恢复整个文件系统或数据卷，无论文件容量或数量如何。



== MetroCluster软件定义存储

ONTAP Select MetroCluster软件定义存储 (SDS) 提供增强的保护和经济高效的实施。

如果满足某些最低要求，双节点集群可以在两个位置之间扩展。此架构完美地融合了基于硬件的MetroCluster和单数据中心集群（硬件定义或软件定义）。ONTAPONTAP Select MetroCluster SDS 的要求凸显了软件定义存储解决方案的普遍灵活性，以及它与基于硬件的MetroCluster SDS 之间的差异。无需专有硬件。

与MetroCluster不同， ONTAP Select使用现有的网络基础架构，并支持高达 5 毫秒 RTT 的网络延迟和高达 5 毫秒的最大抖动，总最大延迟为 10 毫秒。最大距离也要求为 10 公里，尽管延迟情况更为重要。市场空间中的分离要求更多地与物理分离有关，而不是实际距离。在某些情况下，这可能意味着不同的建筑物。在其他情况下，它可能意味着同一建筑物中的不同房间。无论实际的物理位置如何，双节点集群定义为MetroCluster SDS 的原因是每个节点都使用单独的上行链路交换机。

作为双节点 HA 配置的一部分，需要一个调解器在故障转移期间正确识别活动节点，并避免任何裂脑情况（即两个节点在网络分区期间保持独立活动状态）。此操作与之前提供的常规双节点 HA 配置相同。为了在站点故障期间提供适当的保护和故障转移，调解器应与两个 HA 节点位于不同的站点。调解器与每个ONTAP Select节点之间的最大延迟不能超过 125 毫秒。

ONTAP Select MetroCluster SDS 具有以下优势：

* MetroCluster SDS 为ONTAP Select提供了另一个维度（数据中心到数据中心）的保护。现在，除了利用软件定义存储和ONTAP的所有优势之外，您还可以利用这一额外的保护级别。
* MetroCluster SDS 提供业务关键型数据保护，具有 0 RPO 和自动故障转移功能。数据存储和应用程序访问点均可自动切换到正常运行的数据中心或节点，无需 IT 人员干预。
* MetroCluster SDS 经济高效。它利用现有的网络基础架构，在 HA 对之间实现扩展弹性，无需额外硬件。它还能在同一集群中提供主动/主动数据访问和数据中心冗余。
